Get started with npm-updater in seconds. Here are the most common commands you'll use:

ğŸ“¦ Package Manager Support

npm-updater works with all major JavaScript package managers. Use the --pm flag to specify your preferred package manager.

npm

Default

The most widely used package manager for JavaScript. Pre-installed with Node.js. 

# Use npm (default)
npm-updater ls
npm-updater update prettier

pnpm

Fast

Fast, disk space efficient package manager that uses hard links and symlinks. 

# Use pnpm
npm-updater --pm pnpm ls
npm-updater --pm pnpm update prettier

Yarn

Classic

Fast, reliable, and secure dependency management. Available in classic and modern versions. 

# Use Yarn
npm-updater --pm yarn ls
npm-updater --pm yarn update prettier

Bun

Ultra Fast

Incredibly fast JavaScript runtime, package manager, and test runner. 

# Use Bun
npm-updater --pm bun ls
npm-updater --pm bun update prettier

ğŸ”„ update Command

Updates a specific package to its latest version (or a specified version) using the specified package manager.

Syntax

npm-updater [--pm ] update [@version]

Aliases

  • upgrade - Alternative command name
  • -u - Short flag
  • --update - Long flag

Examples

# Update to latest version
npm-updater update prettier
npm-updater --pm pnpm update typescript
npm-updater --pm yarn update eslint

# Output:
Updating prettier to 3.1.1 using npm...
âœ… prettier updated successfully!
# Update to specific version
npm-updater update react@18.2.0
npm-updater update typescript@5.0.0
npm-updater --pm pnpm update @vue/cli@5.0.8
npm-updater --pm yarn update @angular/cli@16.2.0

# Update scoped packages to specific versions
npm-updater update @types/node@20.8.0
npm-updater update @babel/core@7.23.0
npm-updater update @eslint/js@8.52.0

# Update with semver ranges
npm-updater update "react@^18.0.0"
npm-updater update "typescript@~5.0.0"
# Update to special versions
npm-updater update @google/gemini-cli@nightly
npm-updater update experimental-tool@dev
npm-updater update next@canary

# Common distribution tags
npm-updater update react@next
npm-updater update react@beta
npm-updater update react@alpha
npm-updater update node@latest
npm-updater update typescript@beta

# Scoped packages with special versions
npm-updater update @types/node@nightly
npm-updater update @babel/core@next
npm-updater update @vue/cli@beta
npm-updater update @angular/cli@rc
npm-updater update @testing-library/dom@next

# Experimental and development versions
npm-updater update vite@experimental
npm-updater update eslint@dev
npm-updater update prettier@beta
npm-updater update webpack@alpha
Tip: You can specify any version format including semver ranges, dist-tags like latest, next, beta, or special versions like @nightly, @dev, @preview.

ğŸš€ updateall Command

Scans all global packages and updates them to their latest versions using the specified package manager. Automatically detects and handles conflicts.

Syntax

npm-updater [--pm ] updateall

Features

  • Automatically detects all global packages
  • Compares versions with package registry
  • Updates packages sequentially to avoid conflicts
  • Supports special versions (nightly, dev, preview)
  • Provides detailed progress feedback
  • Handles errors gracefully

Examples

# Update all packages using npm (default)
npm-updater updateall

# Update all packages using pnpm
npm-updater --pm pnpm updateall

# Update all packages using Yarn
npm-updater --pm yarn updateall

âš™ï¸ Advanced Usage

Batch Operations

Combine multiple commands in a single script for automated workflows: 

#!/bin/bash

# Check for outdated packages
echo "Checking for outdated packages..."
npm-updater ls

# Update all packages
echo "Updating all packages..."
npm-updater updateall

# Verify updates
echo "Verifying updates..."
npm-updater ls

CI/CD Integration

Use npm-updater in continuous integration pipelines:

# GitHub Actions example
name: Update Global Packages
on:
  schedule:
    - cron: '0 0 * * 0' # Weekly on Sunday

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install npm-updater
        run: |
          git clone https://github.com/involvex/npm-global-updater.git
          cd npm-global-updater
          npm install
          npm run build
      - name: Update global packages
        run: |
          cd npm-global-updater
          ./bin/npm-updater.js updateall

Error Handling

Handle errors gracefully in your scripts:

#!/bin/bash

# Update with error handling
update_package() {
    local package=$1
    echo "Updating $package..."
    if npm-updater update "$package"; then
        echo "âœ… Successfully updated $package"
    else
        echo "âŒ Failed to update $package"
        return 1
    fi
}

# Update specific packages
packages=("prettier" "typescript" "eslint")

for package in "${packages[@]}"; do
    update_package "$package" || echo "Continuing with next package..."
done
