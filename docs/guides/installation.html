Installation Guide

Learn how to install npm-updater and get started with managing your global npm packages efficiently.

üìã Requirements

Before you begin: npm-updater requires Node.js and a package manager (npm, pnpm, Yarn, or Bun) to be installed on your system.

Required Software

  • Node.js (version 14.0.0 or higher) 
    node --version
  • Package Manager (choose one):
    • npm (comes with Node.js)
    • pnpm (recommended for performance)
    • Yarn (classic or modern)
    • Bun (recommended for development)

Optional but Recommended

  • Git - for cloning the repository
  • Bun - for faster development experience

‚ö° Method 1: NPX (Quick Usage)

Use npm-updater directly without installation. Perfect for trying it out or using it occasionally.

1

Run with NPX

# List global packages
npx @involvex/npm-global-updater@latest ls

# Update a package
npx @involvex/npm-global-updater@latest update prettier

# Check latest version
npx @involvex/npm-global-updater@latest latestversion react

‚ú® Benefits of this method:

  • No installation required
  • Always uses the latest version
  • Quick for testing and occasional use
Note: NPX will download and cache the package on first run, which may take a few seconds.

üöÄ Method 2: Clone Repository (Recommended for Development)

This method gives you the most control and allows you to contribute to the project. Ideal for developers who want to modify or contribute to npm-updater.

2

Clone the repository

git clone https://github.com/involvex/npm-global-updater.git
cd npm-global-updater
3

Build the project

bun run build
4

Test the installation

npm-updater version

‚ú® Benefits of this method:

  • Full access to source code
  • Ability to modify and contribute
  • Latest features and updates
  • Development tools included

üì¶ Method 3: Global Installation

Install npm-updater globally on your system for persistent access. This method requires the package to be published to npm registry.

1

Install globally (when published) 

# Using npm
npm install -g @involvex/npm-global-updater

# Using pnpm
pnpm add -g @involvex/npm-global-updater

# Using yarn
yarn global add @involvex/npm-global-updater

# Using bun
bun add -g @involvex/npm-global-updater
2

Verify installation

npm-updater version

‚ú® Benefits of this method:

  • Available system-wide
  • Faster startup (no download required)
  • Works offline after installation
Publishing Status: The package is currently in development. For now, use Method 1 (Clone Repository) or Method 2 (NPX) to try it out.

üîß Post-Installation Setup

Add to PATH (Optional)

If you want to use npm-updater from anywhere after building: 

# Add the bin directory to your PATH
export PATH="$PATH:$(pwd)/bin"

# Add to your shell profile for persistence
echo 'export PATH="$PATH:$(pwd)/bin"' >> ~/.bashrc  # or ~/.zshrc
source ~/.bashrc  # or source ~/.zshrc

Create a Symlink (Alternative)

# Create a global symlink
bun link

Test Installation

Verify that npm-updater is working correctly:

# Show version
npm-updater version

# Show help
npm-updater help

# List global packages
npm-updater ls

üêõ Troubleshooting

Command not found

# Solution: Add to PATH or use absolute path
export PATH="$PATH:$(pwd)/bin"
npm-updater 

# Or use bun run for development
bun run dev

Permission denied errors

# Fix npm permissions
sudo chown -R $(whoami) ~/.npm

# Or use a version manager like nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install node

TypeScript compilation errors 

# Run type checking
bun run typecheck

# Fix formatting and linting
bun run prebuild

Build fails

# Clean install dependencies
rm -rf node_modules bun.lock
bun install

# Run build again
bun run build
Need more help? Check our GitHub Issues or start a Discussion.
